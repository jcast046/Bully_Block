Glossary

  • Alert System: A feature that sends real-time notifications to school staff, administrators, and security personnel upon detection of bullying behavior, facilitating timely intervention and support.

  • Beautiful Soup (Python): A Python library designed for parsing HTML and XML documents, enabling easy extraction and manipulation of data from web pages.

  • BullyBlock: A system developed to monitor, detect, and address cyberbullying behaviors within the Canvas Learning Management System (LMS), providing educators with essential tools to enhance student safety.

  • Canvas LMS: A widely adopted Learning Management System (LMS) utilized in K-12 education that integrates with BullyBlock to effectively monitor and identify instances of cyberbullying among students.

  • CI/CD: Continuous Integration and Continuous Delivery practices implemented using GitHub Actions, a tool that automates the process of testing and deploying applications, ensuring high-quality software delivery.

  • Data Encryption: A security measure employed within BullyBlock to protect sensitive data, ensuring that only authorized users can access the stored information and maintaining confidentiality.

  • Data Privacy Compliance (FERPA): The adherence to the Family Educational Rights and Privacy Act, which mandates that student data is managed securely and utilized strictly within legal parameters to protect student privacy.

  • Data Scraping: The use of algorithms to extract and analyze data from Canvas interactions, enabling the collection of valuable insights for improving the detection of cyberbullying.

  • Dashboard: A visual interface within BullyBlock that consolidates data and analytics related to flagged incidents, assisting educators in reviewing and managing cases of bullying effectively.

  • Documentation Tool: A software application or platform designed to create, manage, and publish documentation for APIs, ensuring that developers can understand and effectively use the features and endpoints provided.

  • False Positives: Instances in which non-threatening interactions are incorrectly identified as cyberbullying, leading to unnecessary alerts and the need for ongoing refinement of detection algorithms.

  • Frontend: The client-side component of an application, developed using React, which is a JavaScript library for constructing user interfaces that enhance user interaction and experience.

  • GitHub Actions: A CI/CD tool that automates the testing and deployment processes for the BullyBlock application, streamlining the development workflow and enhancing software quality.

  • IDE: Integrated Development Environment, such as Visual Studio Code, which provides comprehensive tools for writing, debugging, and managing code in software development.

  • Keras (Python): A high-level neural networks API that simplifies the building and training of complex deep learning models, running on top of TensorFlow to facilitate machine learning development.

  • Machine Learning (ML): A subset of artificial intelligence that improves the cyberbullying detection algorithm by analyzing historical data, refining its ability to accurately identify and respond to instances of bullying.

  • Machine Learning Model: A predictive model that utilizes historical data to enhance the accuracy of cyberbullying detection over time, continuously learning from new data inputs.

  • MongoDB: A NoSQL database selected for its flexibility in managing unstructured data, utilized to store user messages, posts, incidents, and alerts generated by the BullyBlock system.

  • NLP (Natural Language Processing): A technology leveraged by BullyBlock to analyze text data in real-time, enabling the detection of harmful language and bullying patterns in online communications.

  • NLTK (Natural Language Toolkit): A comprehensive Python library for processing and analyzing human language data, supporting tasks like tokenization and part-of-speech tagging to aid in bullying behavior detection.

  • Performance Monitoring Tool: A software application or system that continuously tracks and assesses the performance of an application or service, providing insights into operational efficiency and helping to identify and resolve issues as they arise.

  • Prometheus: A monitoring tool utilized in the project to track system metrics and operational efficiency, helping to identify and resolve issues as the platform scales.

  • Puppeteer (Node.js): A Node.js library used for web scraping, particularly effective for extracting dynamic content rendered by JavaScript on web pages.

  • Real-time Monitoring: The capability of BullyBlock to continuously scan communications as they occur, allowing for immediate detection and alerting of potential cyberbullying incidents.

  • Scalability: The ability of the system to accommodate growth in users or data volume without compromising performance, making it suitable for various school sizes and districts.

  • Scrapy (Python): An open-source web crawling framework designed for large-scale web scraping, enabling efficient data extraction from websites.

  • Sentiment Analysis: A technique within NLP that assesses the emotional tone of text, aiding in the identification of negative or harmful interactions among users.

  • Stanford NLP: A suite of natural language processing tools available in Python and Java, offering capabilities such as named entity recognition, which enhances the accuracy of identifying complex user interactions.

  • TextBlob (Python): A Python library that provides a simple API for performing common natural language processing tasks, including quick sentiment analysis.

  • Testing Frameworks: Tools like Jest and Mocha used for writing and executing tests in JavaScript applications to ensure code quality and functionality.

  • UI: User Interface, the part of the application where users interact with the system, designed for intuitive navigation and functionality.

  • UX: User Experience, the overall perception and satisfaction of users when interacting with the application, focusing on usability and enjoyment.

  • Version Control: A system like GitHub that manages changes to code, facilitating collaboration among developers and maintaining a history of project modifications.

  • VADER (Python, within NLTK): A specialized tool for sentiment analysis, particularly adept at interpreting sentiments in social media text, including content generated within the Canvas platform.

  • Languages: Programming languages such as Java and JavaScript that are utilized in the development of the BullyBlock application, each serving specific functions within the codebase.
